# 🔒 Security Architecture

> **TL;DR:** API-ключи Яндекс Маркета и RetailCRM **никогда не хранятся на диске**. Они вводятся через Telegram, живут только в RAM, и исчезают при перезапуске. Доступ к боту только у указанного в конфигурации username.

---

## Модель угроз

| Угроза | Как защищено |
|--------|-------------|
| Утечка API-ключей с сервера | Ключи **не хранятся** на диске — только в RAM |
| Рандомный человек управляет ботом | Доступ только по `username` — жёсткая проверка |
| Подмена Telegram аккаунта | Проверка `username`, не `id` (username нельзя подделать) |
| Ключи в логах | Маскированы: `AQVN12...x7z9` |
| Ключи в `.env` файле | Нет `.env` — всё вводится через `/setup` |
| Файл сессии Telegram | Только `TG_BOT_TOKEN` — его можно отозвать за секунду |

## Что находится где

| Локация | Что хранится |
|---------|-------------|
| **config_secure.py** | Только `TG_BOT_TOKEN` и `TG_CHAT_ID` |
| **RAM (память)** | `YM_API_KEY`, `RCRM_API_KEY`, URL, кампании |
| **sent_orders.json** | Только ID заказов (не секреты) |
| **На диске НЕТ** | Никаких API-ключей магазинов |

---

## Команды безопасности

| Команда | Описание |
|---------|----------|
| `/setup` | Пошаговый ввод API-ключей через Telegram |
| `/status` | Показать конфигурацию (ключи маскированы) |
| `/clear` | Мгновенно стереть ВСЕ ключи из памяти |
| `/run` | Запустить проверку |
| `/stop` | Остановить проверку |
| `/interval` | Изменить период проверки |

## Сценарии

### Сценарий 1: Сервер взломали
- **Файлы на диске:** только `TG_BOT_TOKEN` → отзываем в @BotFather за 5 секунд
- **RAM:** утеряна при рестарте, ключей магазинов нет
- **Результат:** минимальный урон

### Сценарий 2: Кто-то нашёл ссылку на бота
- **Проверка username:** указанный username ≠ чужой username
- **Результат:** 🚫 Доступ запрещён, владелец получает уведомление о попытке

### Сценарий 3: Нужно срочно отключить
- Отправь `/stop` — проверка остановлена
- Отправь `/clear` — ключи стёрты из памяти
- Отзови токен в @BotFather — бот мёртв

## Запуск

```bash
# 1. Установить зависимости
pip install -r requirements.txt

# 2. Отредактировать ТОЛЬКО config_secure.py
#    Вписать TG_BOT_TOKEN и TG_CHAT_ID

# 3. Запустить
python bot_secure.py

# 4. В Telegram написать боту:
#    /setup → ввести API-ключи
#    /run   → запустить проверку
```

---

## Чего НЕ делать

- ❌ Не отправляйте API-ключи в обычные чаты — только в приват с ботом
- ❌ Не оставляйте `screen`/`tmux` на чужих серверах с запущенным ботом
- ❌ Не используйте `username` на боте, который видят другие

## Если нужно максимальное: локальный запуск

Самый безопасный вариант — запуск на **своём устройстве**:

```bash
# Можно даже без интернета на устройстве:
# - Raspberry Pi дома
# - Старый ноутбук
# - Любой компьютер, который не выключается

python bot_secure.py
# Бот работает, ключи никуда не уходят
```

Для 24/7 можно купить **Raspberry Pi Zero 2 W** (~2000₽) — мини-компьютер размером с кредитку, потребляет 2 Вт, работает от powerbank. Это твоё железо, твои ключи, никаких чужих сервисов.
